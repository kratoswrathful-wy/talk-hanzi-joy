## 案件個別頁（`src/pages/CaseDetailPage.tsx`）

### 兩排結構（維持）

- **第一排**：返回列表、依狀態的流程按鈕、「新增案件」等——**維持現狀**，不因工具列 id 整併而改成單排。
- **第二排**：案件圖示、`CaseIconUploader`（manager）、狀態／標題區、以及「詢案訊息／Slack 詢案／複製本頁」——**維持為第二排區塊**（與第一排分開）。

### 標題與狀態標籤：上下對調

- 在第二排左側主欄（圖示右側）內，將目前順序 **TitleInput → 狀態列（`CaseStatusBadge` + 提示文案）** 改為 **狀態列 → TitleInput**。
- `isPmOrAbove` 與非 PM+ 兩種分支（約原 1633–1785 行區塊）**皆**做相同對調，避免權限不同時版面不一致。

### 標題可顯示範圍延伸至頁面另一端

- **現況問題**：第二排使用 `flex` + `justify-between`，左側 `flex-1 min-w-0` 與右側 `shrink-0` 三鍵**同一列**，標題僅能使用「剩餘寬度」，視覺上無法用到圖示右側至內容區右緣的完整空間。
- **目標**：`TitleInput`（已具 `w-full`）的父層須佔滿「圖示右側至內容區右緣」的寬度。
- **建議結構**（擇一實作即可）：
  - **Grid**：例如 `grid-cols-[126px_1fr]`，左欄跨列放圖示；右欄由上而下：**靠右**對齊三鍵 → 全寬狀態列 → 全寬 `TitleInput`；或
  - **欄式堆疊**：右欄 `flex flex-col w-full min-w-0`，第一列 `flex justify-end` 放三鍵，下列狀態、再下列標題，皆 `w-full`。
- 重點：**避免**標題與右側三鍵在同一 flex 列內橫向分配寬度。
