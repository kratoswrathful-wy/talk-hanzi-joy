Excel 匯入 — 警語與轉 tag 選項（預覽稿）

本頁僅供版型與互動預覽，未連結實際 CAT。下方「套用預覽」為簡化示意，與正式匯入引擎可能略有差異。實作請依 docs/EXCEL_IMPORT_TAG_WARNING_PLAN.md。

匯入試算表時請注意

目前 CAT 專案支援匯入的試算表格式為：.xlsx（Excel 2007 以後）、.xls（舊版 Excel）。其他副檔名（例如 .xlsm、.ods）尚未列為支援；若使用 Google 試算表請先另存新檔為 .xlsx 再上傳。

  • 可翻譯內容會轉為編輯器內部結構；多數儲存格格式、圖表、圖片可能無法保留。
  • 標記（tag）若前後不一致，之後匯出可能失敗；建議完成後執行品質檢查。

無需勾選同意即可繼續（若實作時改為必勾，再於實際 UI 調整）。

匯入時將下列樣式轉為行內 tag（選填）

建議處理順序：先萃取儲存格富文字，再在純文字上套用下列規則。

範例（勾選時）：Price <x> → 可轉為帶 {n} 的行內標記（示意）
範例：See [note] → 括號區可轉為 tag（示意）
範例：{token} — 須與系統內部 {1} 編號規則並存時，由程式接續編號（正式版行為）。
指字面上反斜線與字母 n 兩個字元，常見於客戶檔；非 Alt+Enter 的儲存格換行。

自訂規則（正則表達式） Regex

每點選一次「新增」多一列；條數可由產品訂上限。欄位內為 regex：錯誤語法將無法預覽。在 regex 裡 \n 常指換行字元；若要對應字面 \n 兩字元可另用上方勾選或使用跳脫寫法（如 \\n，視引擎而定）。

預設規則 — 對照預覽文案

以下為與上方四個勾選對應的靜態說明（不依互動文字變動）。

  • < >：偵測成對尖括號內文，轉為行內 tag。
  • [ ]：偵測成對方括號內文，轉為行內 tag。
  • { }：偵測成對大括號內文；編號與既有佔位符接續。
  • \n 純文字換行標記：偵測字面 \ + n，轉為行內 tag；與儲存格內換行無關。
