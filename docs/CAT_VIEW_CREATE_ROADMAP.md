## 3. 產品決策（待實作；對話定案）

以下為**尚未寫入** `cat-tool` 之流程設計，供實作與驗收對照。

當使用者在建立句段集對話框選擇 **「自訂篩選」** 時：

1. 顯示 **「是否預覽」** 核取方塊，**預設不勾選**。
2. 附**警語**（白話）：預覽會載入較多資料，專案或句段很多時**可能較慢**、**較吃裝置效能**；請視需要再開啟。
3. **不勾選「要預覽」**：使用者按下確定後，流程與 **「快速結合」** 相同體感——**直接建立句段集**（不進入預覽專用畫面）。實作時應與現有 [`cat-tool/app.js`](../cat-tool/app.js) `#createViewModal` 確認邏輯銜接；檔案範圍、`filter_summary` 等細節以 PR 為準。
4. **勾選「要預覽」**：使用者按下確定後，進入**預覽專用畫面**（與一般單檔編輯器主畫面分離，避免編輯狀態與 `currentFileId` 等互相干擾）。

**捷徑路徑**：略過預覽畫面即建立時，**§5.2** 所定「步驟二」畫面不出現；篩選條件與 `filter_summary` 之語意以實作與 PR 為準，並應在 SPEC／本文件後續修訂中對齊。

## 4. 分階段實作計畫（建議順序）

可於實作 PR 中逐項勾除。

| 階段 | 目標 | 說明（白話） |
|------|------|----------------|
| **1** | 流程與殼 | 在建立對話框為「自訂篩選」加上「是否預覽」核取方塊與警語。不勾：確認後直接建立（沿用／收斂現有建立路徑）。勾選：確認後只先進入**預覽專用畫面**（可先為空殼或僅標題／返回／建立）。 |
| **2** | 預覽有內容 | 預覽頁載入句段，顯示**簡版唯讀表格**（必要欄位）；視需要加**筆數上限**或提示，避免一次載入過大。 |
| **3** | 對齊 §5.2 完整體驗 | 預覽區上方為與編輯器一致之**完整篩選／搜尋列**，變更條件時**即時**更新下列；**`filter_summary`** 與是否改為**後端依條件查句段**於 PR 註明。 |

本表不鎖定函式名或行號，以免程式重構後文件過期。
