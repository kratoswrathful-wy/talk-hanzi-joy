② 「拆分」直接預填指派表 流程接通

現況：「拆分」只計算工作量均分、不送出指派。新設計：拆分算完直接把建議列範圍與人選預填到上方表格，PM 微調即送出。

操作流程
  1. 選好份數與依據 → 按「計算並預填」。
  2. 系統把建議列範圍寫進①的表格（角色預設「翻譯」、人員留白待選）。
  3. PM 補上每列人員、必要時調整列範圍與交期 → 儲存。

④ 新舊對照

現況 平面人員勾選

  • 無法分翻譯／審稿
  • 無法指定列範圍（誰做哪一段）
  • 無交期
  • 寫入 cat_file_assignments，與 LMS 不同表

新設計 協作式表格

  • 每列可選翻譯／審稿角色
  • 可填列範圍（1-3063、4298-、整檔）
  • 可設交期
  • 統一寫入 cat_stage_assignments，與 LMS 共用
  • 「拆分」可一鍵預填
  • 顯示每列即時進度

⑥ 資料模型：譯者 UUID 雙寫配對 穩定性

選人時同時保留「顯示名稱」與「帳號編號（UUID）」。畫面顯示名字，系統配對認編號。改名、同名都不會讓指派消失。

欄位現況新設計
協作列譯者translator: "威儀"（只有名字）translator: "威儀" ＋ translator_user_id: "8f3a…"
同步配對依據用名字反查（改名／同名即失敗）優先用 UUID；配不到才退回名字
配對失敗時靜默跳過，指派消失回傳失敗名單，PM 收到 toast 提示

UUID 其實在選單裡本來就有（options[].id），目前選定時被丟棄；本案改為一併保留，工程量小、風險低。
