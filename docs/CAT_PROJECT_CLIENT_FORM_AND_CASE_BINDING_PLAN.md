# CAT：專案客戶表單連結與檔案綁定 LMS 案件 — 計畫摘要

> 狀態：**規格已收斂**，對應靜態預覽見使用者「下載」資料夾（檔名見文末）。**尚未實作程式**。

本計畫整理截至目前討論結果：在 **CAT 專案／檔案層**補欄位，並在 **編輯器工具列**依條件顯示按鈕；另移除已廢棄之「標籤群組插入」UI。

---

## 一、目標（行為）

### 1. 專案層：客戶提問表單連結

- **對象**：PM（或有權編輯專案設定者）。
- **欄位**：選填；格式為合法 **HTTPS（或 HTTP）URL**，指向客戶／外部的提問表單。
- **若已填寫**：編輯器工具列顯示 **「客戶表單」** 按鈕；點擊以 **新分頁** 開啟該 URL（建議 `target="_blank"`、`rel="noopener noreferrer"`）。
- **若未填寫**：不顯示「客戶表單」按鈕。
- **資料一致性**：團隊線上版以 **伺服端（Supabase）** 為準；同步規則比照既有 `cat_projects` 欄位（需 migration + `cat-cloud-rpc` 白名單映射時再補）。

### 2. 檔案層：對應 LMS「案件」

- **對象**：PM（或匯入／編輯檔案時可操作者）。
- **語意**：此處「案件」指 **TMS `cases`（LMS 案件）**，**不是** CAT 的 `cat_projects`。綁定用途包含：編輯器內 **「內部註記」** 建立時可寫入 **`related_case`／`relatedCase`**（與現有內部註記模型一致；現為以 **案件標題** 字串關聯）。
- **若該檔案已選擇案件**：編輯器工具列顯示 **「內部註記」** 按鈕；點擊後走既有「新建內部註記」流程（細節見 §四）。
- **若未選擇**：不顯示「內部註記」按鈕。
- **選擇 UI**：採討論已定案之 **方案 A**——以現已載入前端的案件清單做 **本機即時篩選**（打字縮小候選），必要時限制最短字數與結果筆數上限，避免列表過長。

### 3. 編輯器：移除「標籤群組插入」

- 移除 **`#btnTagGroupMode`**（工具列 `T⁺`）及其 **title／提示文案**。
- 一併清理 **`cat-tool/app.js`** 內相關事件、`tagGroupInsertMode` 等死碼路徑（實作時以程式搜尋為準），避免殘留無效邏輯。
- 修改後依 **`npm run sync:cat`** 同步 `public/cat`。

## 二、與先前討論的對齊

| 議題 | 結論 |
|------|------|
| 內嵌 iframe 開外部站 | 不採；外部表單以新分頁開啟即可。 |
| `relatedCase` 用途 | 既有欄位，表示內部註記與哪個 **案件標題** 關聯；非「客戶表單 URL」。 |
| 案件搜尋 | **方案 A**：本機篩選，不做每次打字打後端（方案 B）。 |
| 資料放哪 | 客戶連結：**專案**；案件綁定：**檔案**。 |

---

## 三、尚待在實作前拍板的細節（可選）

1. **「內部註記」按鈕**：在 CAT 為 iframe 時，是否由 **父頁面** 執行 `navigate`／`window.open`／`postMessage`；與現行 LMS 案件頁「新增後進入內部註記編輯頁」一致否。
2. **綁定修改**：檔案綁錯案件時，是否允許在專案檔案清單內 **重新選擇**。
3. **權限**：誰可編輯專案 URL、誰可為檔案選案件。

## 六、驗收（草案）

1. 專案未填 URL → 編輯器無「客戶表單」；填寫並儲存後 → 出現且新分頁開啟正確。
2. 檔案未綁案件 → 無「內部註記」；綁定後 → 出現且可建立／跳轉內部註記（依實作）。
3. 工具列不再出現「標籤群組插入」（`T⁺`），快捷鍵說明無残留誤導文案。
