# Ctrl+1～9 快捷鍵細部規格（實作依據）

**狀態**：產品決策文件（2026-05）；**已實作**（**`c4ef6fe`**）。驗收記錄見 [`docs/CAT第四波主記錄.md`](CAT第四波主記錄.md) **§九點一**。

**關聯**：與 [`docs/CAT第四波主記錄.md`](CAT第四波主記錄.md) **§5-g G4**（列表來源）並存；**細部套用規則**以本文件與該主記錄 **§5-i** 為準。介面用語仍遵守 [`docs/CAT_VIEW_SPEC.md`](CAT_VIEW_SPEC.md) §1.3（使用者可見文案避免「匹配」等簡中慣用詞）。

---

## 1. 目的

在檔案編輯檢視開啟時，**Ctrl+1～9** 依右側面板**作用中分頁**與**第 N 筆**列表項目，將建議內容寫入適當位置；並依**焦點**、**內容類型（TM／TB／Frg）**、**真／假游標**分流，使行為可預測、可驗收。

---

## 2. 術語

| 用語 | 說明 |
|------|------|
| **TB** | 術語庫比對出來的條目（CAT 分頁列表之一類）。 |
| **Frg** | 翻譯記憶中**片段相符**；在 CAT 分頁列表顯示時使用的簡稱。 |
| **TM**（本文件） | **完整**翻譯記憶建議（含 TM 搜尋 concordance 結果，及 CAT 列表中屬「整段 TM 建議」之列）：套用時**整段取代**目標譯文格內容（見 §4）。 |
| **真游標** | 鍵盤焦點在譯文編輯區（`.col-target` 內可編輯區）時之 caret。 |
| **假游標／暫存游標** | 離開譯文格後顯示之靜態標示與／或 `catSavedCaret` 等與假游標同源之記錄（與 [`cat-tool/app.js`](../cat-tool/app.js) 既有實作對照）。 |

**TB／Frg 與 TM 的行為差異（產品定義）**

- **TM**：**整段取代**目前目標譯文格之內容（並依既有邏輯更新 tag 顯示、相符度等）。
- **TB、Frg**：**僅插入**對應之建議譯文文字至游標邏輯位置，**不**以整欄全文取代。

**與程式 `type` 欄位之銜接**：實作時須確認列表資料中 **Frg** 是否與 **TM** 共用同一 `type` 字面。若共用，需依**列語意**或**額外欄位**區分「整段 TM」與「片段 Frg」，並使 **Frg 與 TB 同路徑**（僅插入）。若 Frg 已為獨立 `type`，則直接對應「僅插入」分支。

---

## 3. 列表來源（與 §5-g G4 一致）

- 右側 **TM 搜尋**分頁為作用中（`#tabTmSearch` 等同類 active）：**Ctrl+N** 對應 **TM concordance** 列表第 **N** 筆（當頁 1～9 編號與列表一致處，由實作對照 `applyTmConcordanceAtIndex`）。
- 右側 **CAT**分頁為作用中：**Ctrl+N** 對應 **CAT 比對建議**列表第 **N** 筆（逾 9 筆時分頁，**Ctrl+1～9 對應當頁**，與既有 CAT 表行為一致）。
- 該分頁**無**對應列表或第 N 筆不存在：**不動作**。

---

## 4. 目標句段與插入位置：以真／假游標為準

決定**要寫入哪一句譯文格**以及（**TB／Frg** 時）**字元插入位置**時，**優先**依：

1. **真游標**所在之列與偏移（焦點在譯文編輯區內且可還原位置時）。
2. 否則依**假游標／暫存游標記錄**所指之列與偏移。

**不**僅以「目前列高亮（`active-row`）」覆寫上述定位；若舊程式僅查 `active-row`，實作時應改為與本節一致並於 PR 註明。

---

## 5. 依焦點分流

### 5.1 焦點在譯文編輯區（真游標在譯文格）

- **TM**：對**該格**執行**整段取代**；取代後游標在**插入內容之後**（通常即譯文格末尾，與既有「整段套用」語意一致）。
- **TB／Frg**：在**真游標位置**插入建議譯文；插入後游標在**插入片段之後**。

### 5.2 焦點在篩選／搜尋工具列之「尋找」或「取代」輸入框

實作時以目前 DOM 為準；至少包含 [`cat-tool/index.html`](../cat-tool/index.html) 之 **`#sfInput`**（尋找）、**`#sfReplaceInput`**（取代為…）。

- **TM**：**不**寫入譯文、**不**寫入搜尋框；快捷鍵視為**無效**。在**游標附近**（輸入框之 caret 附近）**短暫顯示**工具提示：  
  **「目前游標在搜尋工具列，無法插入 TM 比對／搜尋結果」**  
  （必要時可微套用字，但須符合 CAT_VIEW_SPEC §1.3。）
- **TB／Frg**：以**純文字**插入目前焦點輸入框之**游標處**；插入後游標在插入內容之後。**不**變更譯文格。

### 5.3 焦點既不在譯文格也不在 5.2 之輸入框

- 套用邏輯與 **§5.1** 相同（TM 整段取代、TB／Frg 插入），但**目標列與位置**改依 **§4** 之**假游標／暫存游標**。
- 完成後，**真游標**應回到**該列譯文格**之對應字元位置（插入後在插入段之後）。

---

## 6. 失敗時（錯誤訊息）

若無法完成插入／取代，**不**變更內容，並向使用者顯示**錯誤訊息**（toast／modal 等形式由實作選定），白話說明至少涵蓋下列原因之一：

- 真／假游標所指之**目的列**因篩選等原因**未顯示**（隱藏）；
- 該句段**不可編輯**（鎖定、唯讀、權限等）；
- **找不到**有效之游標或暫存游標記錄。

---

## 7. Undo／相符度（不得破壞既有矩陣）

凡變更譯文之路徑，仍須符合 [`docs/CAT第四波主記錄.md`](CAT第四波主記錄.md) **§5-g G3**、**§5-h H4**：**Ctrl+Z／Y**、**相符度**（`matchValue`、`applyMatchCellVisual`）與雙擊／F8 等路徑層級一致。本文件不重複技術細節。

---

## 8. 瀏覽器快捷鍵與 `preventDefault`

於編輯器頁內，若成功處理 **Ctrl+數字** 並意圖避免與瀏覽器「切換分頁」等預設行為衝突，可於 `keydown` 對該次事件 **`preventDefault()`**。**不**保證作業系統、網址列、外掛或其他應用程式層級之快捷鍵皆無衝突。

## 9. 實作時建議對照之程式入口（現況）

以下僅供定位，實作後可能更名或重構：

- `applyNumpadMatchForActiveTab`、**Ctrl+1～9** 綁定處
- `applyCatMatchAtIndex`、`applyTmConcordanceAtIndex`
- `handleCatResultApply`（目前多依 `active-row`；與 §4 差異須收斂）

變更僅允許在 [`cat-tool/`](../cat-tool/) 進行，並執行 **`npm run sync:cat`** 同步 [`public/cat/`](../public/cat/)（見根目錄 [`AGENTS.md`](../AGENTS.md)）。

## 10. 驗收要點（摘要）

| 情境 | 預期 |
|------|------|
| TM 搜尋分頁 + 列表有第 N 筆 + §5.1 | 整段取代該句譯文 |
| CAT 分頁 + TB／Frg + §5.1 | 於游標插入，不整欄取代 |
| 焦點在 `#sfInput` 或 `#sfReplaceInput` + TM | 不寫入，顯示 §5.2 工具提示 |
| 同上 + TB／Frg | 純文字插入該輸入框游標處 |
| §5.3 + 假游標有效 | 與 §5.1 類似，目標依假游標；完成後真游標回譯文格對應處 |
| §6 任一失敗條件 | 不寫入，顯示錯誤說明 |
