# CAT 分頁 UIUX：置頂資訊列、TM Top5、TB 灰底線與即時 missing（方案 B）

> **更新日期**：2026-05-28  
> **範圍**：CAT 右欄「CAT」分頁（比對結果列表）與編輯器原文欄 TB inline 提示  
> **程式**：`cat-tool/app.js`、`cat-tool/style.css`（同步 `npm run sync:cat` → `public/cat/`）

---

## 1. 需求背景（白話）

使用者在 CAT 分頁檢視 TM/TB 比對結果時，希望：

- **分頁資訊列**能固定在 CAT 分頁最上方，與 `# / 原文 / 譯文` 表頭一起凍結。
- **頁數超過一頁（≥10 筆）**時才顯示左右翻頁控制與快捷鍵提示（避免數字鍵不足）。
- 原文欄 TB 底線能同時顯示：
  - **目前 CAT 分頁可見（當頁 1–9）**的 TB（橘色底線 + 上標 1–9）
  - **有命中但不在當頁可見**的 TB（灰色底線）
- 「TB 是否正確引用（missing）」在譯文輸入時要更即時，而且避免每次都重包 DOM（方案 B）。

---

## 2. 定案行為

### 2.1 右欄 CAT 分頁：置頂資訊列（Sticky）

- 只在右欄 **CAT 分頁**顯示，視覺上屬於 CAT 表格的一部分。
- 與 `# / 原文 / 譯文` 表頭 **一起凍結**在結果列表滾動區的最上方。

顯示內容：

- **總筆數 1–9**：顯示 `比對結果共 x 筆`
- **總筆數 ≥10**：顯示 `比對結果共 x 筆，第 y/z 頁`，並以細直線分隔顯示
  `快捷鍵：◀（目前設定值）/（目前設定值）▶`

分頁：

- 每頁固定 **9 筆**；因此第 10 筆起在第 2 頁。
- 翻頁支援：
  - 點擊上一頁/下一頁按鈕
  - 自訂快捷鍵 `catResultPagePrev` / `catResultPageNext`

### 2.2 TM 顯示規則（Top 5）

- TM 命中中，**相符度最高的 5 筆**置頂顯示。
- 第 6 筆起的 TM **移到整份清單最後**（在所有 TB / Fragment 後）。
- 同分數 tie-break：`lastModified` 優先於 `createdAt`（越新越前）。

### 2.3 原文欄 TB inline：可見橘色 / offpage 灰色

在原文欄的 TB inline 底線與上標（**相對目前右欄當頁**，非永久綁定「第 10 筆以後」）：

- **可見（當頁 9 格內的 TB 列）**：橘色底線 + 上標 1–9
- **offpage（不在當頁的比對列）**：灰色底線（不加上標）

換頁後雙向對調：例如共 15 筆時，第 1 頁看列 10–15 為灰；切到第 2 頁後列 10–15 變橘+上標，原列 1–9 改灰、上標消失。

### 2.4 TB missing 即時更新（方案 B）

原則：

- 第一次 decorate 時會包上 `.tb-inline-term` / `.tb-inline-sup-anchor` 並寫入必要的 `data-*`。
- 譯文每次 `input`（非 IME 組字中）只做：
  - 重新判斷是否命中 `termMatches()`
  - 對既有 DOM **切換 `is-missing` class**
- 不做重包文字、不 rebuild、不 unwrap（降低卡頓風險、提高即時性）。

---

## 3. 驗收清單

### 3.1 置頂資訊列與凍結

1. 右欄切到 CAT 分頁：資訊列位於表頭下方，且隨列表捲動時保持固定。
2. 切到「TM 搜尋 / 新增術語 / QA」：資訊列不顯示。
3. 總筆數 1–9：只顯示 `比對結果共 x 筆`。
4. 總筆數 ≥10：顯示頁碼、左右按鈕與快捷鍵提示（含箭頭圖示）。

### 3.2 TM Top5

1. TM 命中 ≥6：前 5 筆置頂；其餘 TM 出現在整份清單最後。
2. 同分數：依 `lastModified`（無則 `createdAt`）越新越前。

### 3.3 TB 灰底線 + missing 即時更新

1. 原文欄：當頁 TB 命中為橘底線 + 上標；offpage 命中為灰底線（無上標）。
2. **副行**：當頁 TB 顯示 `N 原文→譯文`；offpage TB 亦列出譯法但**無 N**、膠囊偏淡。
3. **換頁對調**（總筆數 ≥10）：第 2 頁時原第 1 頁列改灰、上標消失；切回第 1 頁應恢復。
4. 在譯文欄補上/刪掉 TB 對應譯法：原文欄 missing 樣式能快速更新，不需切句段或換頁。

---

## 4. 已知限制（目前設計的取捨）

- TB inline 目前仍以「不跨 tag-pill / 元素邊界」為前提，只處理 text nodes；跨節點的詞不會合併比對。
- offpage 灰底線表示「有命中但不在**目前右欄當頁** 9 格內」；換頁後橘/灰與上標會對調，不提供 offpage 上標（避免與數字鍵混淆）。
