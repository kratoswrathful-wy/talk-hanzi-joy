mqxliff 進階篩選：確認身分（T／R1／R2）與未確認 — 排版預覽

本頁為靜態視覺預覽，不連結實際 cat-tool。預設假設僅在 .mqxliff 開檔時顯示「確認身分」相關勾選；與現有「已確認／未確認」並列時的語意請見各變體說明。

變體 A — 扁平第三列（與現有 row1／row2 相同 grid 語感）最接近現狀擴充

  • 實作成本低：在 sf-adv-status-row2 前（或後）插入 sf-adv-status-row-mq 即可。
  • 建議加一行 role-note 說明：勾選 T／R1／R2 時僅篩「已確認」句段（與 evaluateSegment 第四維度一致）。
句段狀態

已確認句段：篩選由誰按下最終確認（與格線狀態欄圖示相同）。未確認句段不受此列勾選影響。

變體 B — 左色條子區塊（memoQ 身分視覺分群）可讀性佳

  • 與「一般狀態」用邊界區隔，較不易誤以為在篩 Key 內的角色名。
  • 小螢幕仍建議整塊在 mqxliff 時才顯示。

變體 C — 圖示 pill（與格線狀態欄強對齊）辨識最快

  • 勾選區較佔寬度；適合寬工具列或進階面板全寬時。

變體 D — 窄欄＋橫向捲動（工具列極窄時）防換行碎裂

  • 外層 max-width 模擬側欄或窄視窗；內列 flex-wrap:nowrap 避免「✓+」斷行。
句段狀態

若採此變體，建議仍將「內容／確認／鎖定／memoQ 身分」分維度實作，僅畫面擠在同一捲列。
