## 1. 產品目標（白話）

- **路線**：採用與 Phrase（Memsource）類似的說明方式——匯入時將 Excel **轉成內部可翻譯表示**，**不承諾**保留整本試算表之圖表、圖片、多數格式等；介面須有**顯著警語**（**僅列出目前 CAT 實際支援之試算表副檔名**：`.xlsx`、`.xls`；並註明不支援項目如 `.xlsm`、`.ods` 須先另存為支援格式等）。
- **同意勾選**：**不強制**；使用者可不勾選即繼續（預覽稿已反映）。若日後法務要求再改。
- **Tag 規則（僅 Excel、匯入階段）**：
  - 預設勾選：**`<>`**、**`[]`**、**`{}`** 三種括號樣式（內建規則，非使用者手寫 regex）。
  - 勾選：**`\n` 純文字換行標記（非儲存格換行）** — 指字面上反斜線 + `n` 兩個字元。
  - **自訂比對**：位於四個勾選**下方**，「**新增**」一次多一列；條數可產品訂上限（預覽稿為無上限示意）。欄位為 **regex**；旁邊固定提醒「此為正則表達式」。
- **處理順序**：**先**執行既有 **Rich Text** 萃取 → **再**於純文字上套用括號／字面 `\n`／自訂 regex。
- **預覽**：四個預設勾選各有**靜態說明**；另設**互動區**（使用者貼純文字 →「套用預覽」→ 示意結果）。預覽腳本僅供頁面 demo。
- **不在本計畫內**：`cat-tool/app.js` 等程式實作、資料庫 migration、團隊雲端 RPC 變更。

## 4. 風險與取捨

- **`{}` 與內部 `{1}`**：新 tag 編號接續；預覽腳本為簡化版，正式版須排除已產生佔位符之重疊。  
- **Regex 與字面 `\n`**：自訂欄位之 `\n` 語意與「字面兩字元」勾選須在說明區分。  
- **Phrase 路**：不承諾整份 xlsx 物件原封不動。

## 5. 驗收（實作完成後）
